# Fix Background Location Tracking

## Goal
Enable location tracking and syncing to continue when the mobile app is minimized, screen is off, or app is completely closed.

## Current Problem
- Locations stop syncing when app is backgrounded
- GPS collection pauses during minimization
- Local queue is lost if app closes
- No foreground service configured

## Solution Pattern
Implement a background location service that:
1. Runs even when app is minimized/closed
2. Continues collecting GPS locations every 2-5 seconds
3. Persists locations to local storage
4. Syncs batches to `/api/tracking/batch` every 15 seconds
5. Shows foreground notification (required on Android)

## What You Need to Implement

### 1. Persistent Queue Service
- Store locations in AsyncStorage (RN), SQLite, or local DB
- Prevent buffer overflow (max 100 locations)
- Survive app restart and crashes

### 2. Background Location Manager
- Use native background location APIs:
  - **React Native**: `react-native-background-actions` + `react-native-geolocation`
  - **Flutter**: `background_locator` + `geolocator`
  - **Native**: Android ForegroundService, iOS CLLocationManager
- Request BACKGROUND_LOCATION permissions
- Show persistent notification (Android requirement)

### 3. Background Sync Task
- Run HTTP batching every 15 seconds in background
- Implement retry logic with exponential backoff
- Queue failed batches for next sync attempt
- Include battery level, accuracy, speed, heading

### 4. Battery Optimization
- Reduce location accuracy when battery < 20%
- Increase sync interval to 30s on low battery
- Resume normal 15s interval when battery recovers

### 5. Lifecycle Management
- Start background service when trip starts
- Stop service when trip ends
- Handle app termination gracefully
- Reconnect on app resume

## Tech Stack Guidance

**Choose one path based on your framework:**

### React Native Path
```bash
npm install react-native-background-actions react-native-geolocation-service
```

### Flutter Path
```bash
flutter pub add background_locator geolocator workmanager
```

### Native iOS Path
- Use CLLocationManager
- Configure UIApplicationDidEnterBackground

### Native Android Path
- Use ForegroundService
- Use WorkManager for periodic sync
